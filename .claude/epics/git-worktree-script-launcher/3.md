# Task: Build repository selection, validation, and recent repository persistence

## Description
Implement repository picking and validation flows with clear user messaging, and persist recent repositories for fast relaunch behavior.

## Acceptance Criteria
- [ ] User can select a local folder from a file picker.
- [ ] Invalid repository selection shows actionable error messaging and blocks worktree/script features.
- [ ] Valid repository is persisted and automatically restored on app relaunch when path is available.

## Technical Details
- Implement `select_repository` and `validate_repository` command handlers.
- Add UI states for empty, validating, valid, and invalid repository states.
- Persist recent repositories and restore the most recent valid path on startup.
